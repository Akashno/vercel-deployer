The Vercel Paid-Seat Restriction

Vercel blocks non-paid team seats from triggering branch rebuilds or logging in. Here is how we bypass it cleanly.

The Vercel Way
Requires Paid Vercel Seats
Every PM, QA engineer, designer, or client needs a Vercel license ($20+/month per user) just to trigger redeploys.
Direct Account Access
Sharing access requires exposing core Vercel platform configurations, which poses billing and security risks.
The Empty Commit Bypass
Free Self-Hosted Dashboard
Bypasses seats by programmatically committing/pushing a safe, empty commit tree directly using GitHub Git Data APIs.
Clean History, No workflows
Uses git commit --allow-empty which modifies 0 lines, keeping history tidy. No repo actions or workflows required.

Quick Self-Hosting Guide

Deploy this dashboard locally or to any hosting service supporting Node/Docker in less than 5 minutes.

1

Clone the Repository

Clone the project repository to your local system or deployment server, and run package installations.

2

Configure Environment

Create a .env file based on .env.example. Set up Vercel keys and optional GitHub token parameters.

3

Launch Dashboard

Start the developer server with npm run dev or build production outputs using Nuxt Nitro bundling commands.

# Clone the repository
git clone https://github.com/Akashno/vercel-deployments.git
# Change directory
cd vercel-deployments
# Install Node modules
npm install
# Vercel Configuration
VERCEL_TOKEN="vcl_token_here"
PROJECT_ID="prj_projectIdHere"
TEAM_ID="" # Required only if project is in a team
# Dashboard Authentication
AUTH_USERNAME="admin"
AUTH_PASSWORD="dashboardPass123"
AUTH_SECRET="d485e982ca8586cfb9a9108b98124efac6b27" # 32-character hex key
# GitHub Integration (For deploying without paid seat)
GITHUB_TOKEN="github_pat_with_contents_write" # Fine-grained PAT (Contents: read/write)
GITHUB_OWNER="Akashno"
GITHUB_REPO="vercel-deployments"
# Jira Integration (Optional)
JIRA_ORG="acme"
JIRA_EMAIL="user@company.com"
JIRA_API_TOKEN="jira_token_here"
# Start dev environment (runs on port 3000)
npm run dev
# Build for production distribution bundle
npm run build
# Run local preview server
npm run preview
